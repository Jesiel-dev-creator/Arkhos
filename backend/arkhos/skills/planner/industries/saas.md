# SaaS Landing Pages

## Section Order

Hero (headline + subheadline + CTA + product screenshot/demo) → Logos/Social Proof (client logos or user count) → Problem Statement (pain points in 3 bullets) → Solution (how the product solves each pain) → Features (3-6 with icons and descriptions) → How It Works (3-step visual process) → Testimonials (with photos, names, companies) → Pricing (toggle monthly/yearly, 3 tiers) → FAQ (6-8 common objections) → Final CTA (recap value + button) → Footer.

## Copy Patterns

SaaS copy is benefit-driven and specific. Lead with the outcome: "Ship 10x faster" not "A development platform". Use contrast framing: "Stop doing X manually. Start doing X automatically." Pricing copy should emphasize value per tier, not just feature lists. Free tier language: "Free forever" or "No credit card required" builds trust. Enterprise tier: "Contact sales" with a benefit like "custom integrations and dedicated support".

## Key Features to Include

- Product screenshot or animated demo in hero
- Pricing table with monthly/yearly toggle
- Integration logos (Slack, GitHub, Stripe, etc.)
- Security badges (SOC2, GDPR, SSL)
- API documentation link for developer-focused products
- Free trial or demo CTA

## Design Aesthetic

Dark mode preferred: deep navy or void black backgrounds with high-contrast accent colors (cyan, electric blue, or bright green). Clean, geometric layouts. Monospace or technical display fonts paired with clean sans-serif body text. Dashboard screenshots should have subtle glow or shadow effects. Minimal use of illustrations — real product UI is more convincing than abstract graphics.

## Photo Categories

Product screenshots, dashboard mockups, abstract geometric patterns, developer workspace photos, team collaboration images. Avoid generic handshake photos and clip-art-style illustrations. SaaS visitors are sophisticated and expect visual credibility.
