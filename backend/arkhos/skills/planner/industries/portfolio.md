# Developer Portfolio Landing Pages

## Section Order

Hero (name + title + one-line value prop + CTA) → About (brief bio, tech stack, years of experience) → Projects Grid (3-6 case studies with thumbnails, tech tags, and links) → Skills/Tech Stack (visual grid or categorized list) → Experience Timeline (optional, for senior devs) → Testimonials/Recommendations → Contact Form → Footer with social links.

## Copy Patterns

Portfolio copy should be concise and confident without arrogance. First person is appropriate: "I build..." not "We offer...". The hero should state what you do and for whom: "Full-stack engineer building fintech products" not "Welcome to my portfolio". Project descriptions focus on impact: "Reduced load time by 60%" not "Built a React app". Use technical language appropriately — your audience is recruiters and tech leads who understand it.

## Key Features to Include

- Project cards with live demo links and GitHub repos
- Tech stack badges (React, TypeScript, Python, etc.)
- Downloadable CV/resume link
- Contact form with clear response time expectation
- Social links: GitHub, LinkedIn, Twitter/X
- Dark mode toggle (developers expect this)

## Design Aesthetic

Minimal and clean with a single accent color. Dark mode by default — developers live in dark mode. Monospace display font (Space Mono, JetBrains Mono) for headings, clean sans-serif (DM Sans, Inter replacement: Outfit) for body. Grid-based layout with consistent card sizes. Subtle hover animations on project cards. No decorative elements — let the work speak. Code snippets or terminal-style elements add authenticity.

## Photo Categories

Abstract geometric backgrounds, code editor screenshots, workspace flat-lays, architectural/structural photography for visual texture. Avoid selfies and casual photos unless intentionally personal. Project thumbnails should be actual screenshots, not mockups.
